Library Management Project

  • View On Github

    • I created a library management system that keeps track of books currently in the library system, allows Members to check out, return, and reserve books, and allows Librarians to manage library books.

    My goal was to create a library system that is interactive by means of object-oriented programming. Library Members and Librarians are able to interact with our library system in a way that mimics real life interactions. By means of object-oriented programming, I am able to neatly manage various components of the library while prioritizing user-privacy and system security.

    This library system handles user-input according to specific prompts, handling user-error appropriately. Librarians have the ability to add books to the library system and edit their information as desired. This information is stored and written to a CSV file containing all book data, and serialized at various points in the program flow. When requested by a user, catalog and book information can be displayed. Additionally, the library system reads input data from two CSV files: books.csv, which contains well-known book titles along with their information, and users.csv, which keeps track of users affiliated with the library system. When serialized data exists, it is read from books.ser and users.ser.

    All users in the library who have the ability to check out books can only have 5 books checked out at a time. Additionally, they must adhere to a 10 day book checkout period. If the user does not return their checked out books to the library by the tenth day, the book’s status will be set to “lost” and the user’s account status will be set to “hold”. This system uses two enumeration classes to establish book statuses and account statuses. Additionally, there are two interfaces: a Librarian interface and a Member interface. Each interface contains methods pertaining to the three menu types: the main menu, management menu, and reservations menu.
